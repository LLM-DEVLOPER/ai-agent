# 评测、幻觉治理与 bad case

频次：

- `评测 / 指标 / 幻觉治理`：`32`
- 覆盖面经数：`25`

核心问题：

1. RAG / Agent 项目怎么分层评测？
2. 检索效果、回答效果、工具调用效果怎么分别评估？
3. 幻觉的主要来源有哪些？
4. bad case 怎么分类？
5. 离线评测和在线评测如何联动？
6. 如果业务说系统不靠谱，怎么拆成指标？
7. 线上效果退化时怎么定位是模型、知识库还是工具问题？

高频追问：

1. 评测集怎么构造？
2. 自动评测和人工评测怎么结合？
3. 幻觉治理应该加检索、加约束，还是换模型？
4. 评测指标如何映射到业务指标？

回答抓手：

- 评测分层：`召回层 -> 生成层 -> 工具层 -> 业务层`
- 幻觉闭环：`采样 -> 标注 -> 分类 -> 修复 -> 回归`
- 一定把“不靠谱”翻译成可量化指标
