- `RAG / 检索 / 召回 / 重排`：`99`
- 覆盖面经数：`55`

1. RAG 解决什么问题？

5. query rewrite、query expansion、router、rerank 各解决什么问题？

高频追问：

1. 召回率低时优先查什么？
2. 什么时候该改召回，什么时候该改重排？
3. 业务权限和时间过滤怎么接进检索链路？

回答抓手：

- 固定链路：`query 处理 -> 召回 -> 重排 -> 上下文构造 -> 生成 -> 约束`
- 排障顺序：`分块 -> embedding -> 召回 -> rerank -> prompt -> answer constraint`
- 一定讲清“工程排障”而不是只讲概念
