# 检索、召回与向量库

频次：

- 归一化 `向量库 / 检索`：`24`
- 覆盖面经数：`20`

核心问题：

1. 向量数据库如何选型？
2. `pgvector`、`FAISS`、`Milvus` 这类方案各适合什么场景？
3. `IVF` 和 `HNSW` 的区别是什么？
4. BM25 和向量检索各有什么优劣？
5. 为什么很多业务要做混合检索？
6. embedding 模型如何选？
7. 召回和重排的职责边界是什么？
8. 检索层怎么评测？

高频追问：

1. 电商、客服、企业知识库为什么检索策略不同？
2. 向量库如何支持权限和时间过滤？
3. 大规模索引更新怎么做？
4. 检索层 latency 高时优先优化哪里？

回答抓手：

- 先讲数据规模、过滤需求、更新频率
- 再讲索引结构和近似搜索
- 最后讲指标：`召回率、延迟、存储成本`
