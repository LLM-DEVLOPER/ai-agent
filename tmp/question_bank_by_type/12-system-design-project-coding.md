# 系统设计、项目深挖与编码题

项目深挖高频问题：

1. 项目目标是什么？
2. 你真实负责什么？
3. 为什么这么选型？
4. 效果怎么量化？
5. 最难的 bad case 是什么？
6. 上线后出过什么事故？
7. 如果重做一遍会改什么？

系统设计高频题型：

1. 设计一个企业知识库问答 Agent
2. 设计一个客服 Agent
3. 设计一个社区内容问答 / 生成 Agent
4. 设计一个 DeepResearch 类 Agent
5. 设计一个可观测、可灰度、可回放的 Agent 平台

推荐回答结构：

1. 业务目标
2. 数据与知识源
3. 模型、检索、工具、记忆方案
4. 评测与指标
5. 工程化：并发、容错、成本、发布

编码题高频方向：

1. `LRU`
2. 字符串处理
3. 树的序列化 / 反序列化
4. 并发任务调度
5. 基础检索或数组题

准备建议：

- 项目回答一定要有指标和失败案例
- 系统设计题不要一上来就谈模型，先谈目标和约束
- 编码题守住：`哈希、链表、树、堆、并发基础`
