# 工程化、后端服务与并发治理

频次：

- `工程化 / 后端 / 并发 / 服务`：`32`
- 覆盖面经数：`24`

核心问题：

1. 多用户并发下 Agent 状态怎么隔离？
2. 长任务、异步任务、工具调用链怎么编排？
3. 如何做日志、trace、回放和问题定位？
4. 外部工具接口不稳定怎么办？
5. 服务高可用和回滚机制怎么设计？
6. 多轮状态一致性如何保证？
7. 并发任务和工具调用的幂等如何处理？

高频追问：

1. 为什么 Agent 系统更需要回放能力？
2. 如何统计工具成功率、失败率和延迟分布？
3. 如何设计任务队列和重试策略？

回答抓手：

- 拆层：`接口层、编排层、模型层、工具层、观测层`
- 重点讲：`状态隔离、幂等、重试、降级、熔断、审计`
- 有条件就带上 `trace id / session id / tool run id`
