# 多模态与截图理解  (面试题整理版)

## 1. 为什么这一类题会在二面三面出现？

因为多模态题很适合追问你是否真的理解索引、检索、评测和解释链路，而不是只会做文本问答。

## 2. 第一层：多模态基础题

1. 多模态 RAG 和纯文本 RAG 有什么不同？
2. 图片、截图、图表如何建索引？
3. CLIP 适合解决什么问题？
4. 多模态用户信息如何存储和使用？

## 3. 第二层：截图理解题

1. 给定软件截图，如何通过 RAG 帮用户理解界面组件？
2. 组件识别、结构化理解、检索解释之间是什么关系？
3. 图片理解类任务如何和工具调用结合？

## 4. 第三层：评测题

1. 图表类内容应该怎么评测？
2. 伪多模态 RAG 和原生多模态 RAG 的区别是什么？
