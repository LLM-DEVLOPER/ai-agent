# 项目深挖  (面试题整理版)

## 1. 为什么项目深挖是所有技术问题的最终落点？

因为面试官最终不是在考你会不会背知识点，而是在确认：这些知识点你有没有真正落到项目里，项目里你到底做了什么，以及你碰到问题时有没有真实的判断和复盘能力。

## 2. 项目深挖最常见的起手问题

1. 项目目标是什么？
2. 你真实负责什么？
3. 为什么这么选型？
4. 效果怎么量化？
5. 最难的 bad case 是什么？
6. 上线后出过什么事故？
7. 如果重做一遍你会改什么？

## 3. 面试官最喜欢顺着追的方向

1. 数据从哪里来，怎么清洗？
2. 为什么选这个模型和框架？
3. 为什么不用别的方案？
4. 你真正改动的模块是什么？
5. 你有没有失败案例？

## 4. 项目讲述的推荐顺序

1. `业务场景`
2. `我的职责`
3. `系统架构`
4. `关键技术点`
5. `指标和结果`
6. `问题与复盘`

## 5. 最容易被识破的问题讲法

1. 通篇都在说“我们”。
2. 完全没有量化指标。
3. 说不出失败案例。
4. 说不清自己负责哪一块。
