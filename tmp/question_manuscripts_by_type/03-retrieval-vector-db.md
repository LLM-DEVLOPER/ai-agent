# 检索、召回与向量库  (面试题整理版)

## 1. 这类题为什么重要？

很多候选人会说自己做过 RAG，但一旦面试官往下问到 `向量库怎么选`、`为什么用混合检索`、`IVF 和 HNSW 区别是什么`，就会开始发虚。也就是说，这一类题是用来区分“做过 RAG 项目”和“真正理解检索层”的。

## 2. 面试官通常怎么展开？

1. 先问向量数据库如何选型。
2. 再问 embedding、召回、重排的职责边界。
3. 再问 BM25、混合检索、图谱检索的区别。
4. 再问索引更新、过滤、延迟优化。
5. 最后回到业务约束和线上效果。

## 3. 第一层：选型题

1. 向量数据库如何选型？
2. `pgvector`、`FAISS`、`Milvus` 适合什么场景？
3. 向量库应该嵌在主库里还是独立部署？
4. 存储成本和查询延迟之间如何权衡？

## 4. 第二层：索引结构题

1. `IVF` 和 `HNSW` 的区别是什么？
2. 索引构建速度和检索速度一般怎么权衡？
3. 大规模索引更新怎么做？
4. 召回层为什么经常要配合重排层？

## 5. 第三层：检索策略题

1. BM25 和向量检索各有什么优缺点？
2. 为什么很多业务最后会落到混合检索？
3. embedding 模型如何选？
4. 召回和重排的职责边界是什么？
5. 检索 latency 高时优先优化哪里？

## 6. 第四层：业务约束题

1. 权限过滤应该放在哪一层？
2. 时间过滤和业务过滤是怎么进检索链路的？
3. 电商、客服、企业知识库的检索策略为什么不同？
4. 知识图谱、图数据库和向量检索的边界是什么？
