# 工程化、后端服务与并发治理  (面试题整理版)

## 1. 为什么开发岗会特别看这一类题？

因为 AI 应用项目最后还是要上线，而一旦上线，就会遇到并发、幂等、回滚、回放、工具不稳定、状态串台这些非常后端化的问题。

## 2. 第一层：服务化题

1. 多用户并发下 Agent 状态怎么隔离？
2. 长任务、异步任务、工具调用链怎么编排？
3. 多轮状态一致性如何保证？
4. 工具调用如何保证幂等？

## 3. 第二层：观测与排障题

1. 如何做日志、trace、回放和问题定位？
2. 为什么 Agent 系统更需要回放能力？
3. 如何统计工具成功率、失败率和延迟分布？
4. 外部工具接口不稳定怎么办？

## 4. 第三层：高可用题

1. 服务高可用和回滚机制怎么设计？
2. 任务队列和重试策略怎么设计？
3. 熔断、降级、兜底策略分别适合什么场景？
