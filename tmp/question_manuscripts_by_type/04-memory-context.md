# 记忆、上下文工程与多轮对话  (面试题整理版)

## 1. 为什么这类题越来越高频？

因为面试官已经默认你知道上下文窗口有限，所以他们真正要看的是：当上下文有限、轮次很多、用户历史复杂时，你有没有办法让系统继续稳定工作。

## 2. 面试官通常怎么展开？

1. 先问短期记忆、长期记忆怎么设计。
2. 再问上下文为什么不是越长越好。
3. 再问多轮对话状态如何管理。
4. 再问记忆和 RAG 的边界。
5. 最后追到记忆污染、记忆衰退、并发状态一致性。

## 3. 第一层：记忆分层题

1. 短期记忆、长期记忆、工作记忆分别是什么？
2. 会话态、用户态、任务态应该怎么拆？
3. 用户画像类记忆和任务状态类记忆是否要分开存？

## 4. 第二层：上下文管理题

1. 长上下文为什么不是越长越好？
2. 多轮对话上下文裁剪怎么做？
3. 上下文压缩、摘要记忆、原始轨迹保留各适合什么场景？
4. 当 token 预算有限时优先裁什么？

## 5. 第三层：记忆系统题

1. 长期记忆如何写入？
2. 长期记忆如何召回？
3. 长期记忆如何淘汰？
4. 如何做记忆衰退？
5. 如何防止旧信息干扰新任务？

## 6. 第四层：多轮状态与隔离题

1. 多轮状态如何在并发下隔离？
2. 为什么客服类场景容易出现串台问题？
3. 多轮状态一致性如何保证？
4. 记忆和 RAG 的边界是什么？
