# 多 Agent 协作  (面试题整理版)

## 1. 为什么这类题容易被问深？

因为多 Agent 听起来高级，但真正落地时最容易翻车。面试官会用这类题看你是不是“为了显得高级而设计复杂系统”。

## 2. 第一层：价值判断题

1. 什么情况下值得上多 Agent？
2. 多 Agent 和单 Agent 相比，收益是什么？
3. 多 Agent 的代价是什么？
4. 什么场景根本不值得上多 Agent？

## 3. 第二层：设计题

1. 多 Agent 状态共享怎么做？
2. 多 Agent 通信协议怎么设计？
3. 多个 Agent 结果冲突时谁来裁决？
4. 如何避免死循环和重复执行？

## 4. 第三层：工程题

1. 多 Agent 如何做成本控制？
2. 多 Agent 如何做并发控制？
3. 多 Agent 如何做日志、回放和观测？
