## 三、最该优先准备的题型

### P0：一定会被问

1. 什么是 `Agent`，和 `Workflow`、普通聊天应用有什么区别？
2. `RAG` 整体链路怎么搭，分块、召回、重排、生成分别怎么做？
3. `Agent` 的记忆怎么设计？短期记忆、长期记忆、上下文裁剪怎么做？
4. `Function Calling / 工具调用` 怎么设计？失败如何兜底？超时如何处理？
5. 项目深挖：你做过的 `AI Agent / AI应用 / RAG` 项目，架构、难点、指标、线上问题分别是什么？

### P1：高频且拉开差距

1. 向量数据库如何选型？`pgvector / HNSW / IVF / 混合检索` 分别怎么选？
2. `RAG` 怎么评测？召回、准确率、命中率、幻觉率怎么定义？
3. 长上下文怎么处理？压缩、裁剪、摘要、记忆回写怎么做？
4. 推理部署怎么做？`vLLM / KV Cache / FlashAttention / 量化 / 吞吐和延迟 tradeoff`
5. 工程化怎么落地？并发、限流、状态管理、日志观测、故障定位怎么做？

### P2：二面三面常见

1. `Prompt` 怎么设计？意图不完整时怎么补全？什么时候用 `ReAct`？
2. `LoRA / SFT / DPO / PPO / RLHF` 分别解决什么问题？
3. `Transformer / Attention / MHA / MoE / Qwen` 这些基础原理会怎么追问？
4. 多 Agent 怎么协作？角色划分、冲突处理、状态共享怎么做？
5. 手撕题会考到什么程度？常见是 `LRU / 字符串 / 树 / 并发`。

## 四、复习优先级建议

如果你主要面 `AI Agent 开发 / LLM 应用开发`，建议按这个顺序准备：

1. `Agent 基础 + RAG + 项目深挖`
2. `记忆 / 上下文工程 + 工具调用 + MCP`
3. `检索 / 向量库 / 评测 / 幻觉治理`
4. `推理部署 / 性能优化 / vLLM`
5. `工程化服务能力：并发、限流、日志、容错、状态一致性`
6. `Prompt / ReAct / 多 Agent`
7. `LoRA / DPO / PPO / Transformer 基础`
8. `手撕题`

## 五、结论

如果只看这 `100` 篇开发向面经，最核心的结论是：

- 不是只会讲 `Agent` 概念就够了，真正高频的是 `Agent + RAG + 记忆 + 工具调用 + 工程落地`
- 面试官非常爱从 `项目深挖` 切进去，再转到 `RAG、上下文、工具调用、评测、线上问题`
- 到二面三面，明显会继续追 `推理优化、评测体系、并发一致性、LoRA/RLHF、Agent 框架差异`

所以真正的主线不是：

- 只背模型八股

而是：

- `会做 Agent 应用`
- `会讲 RAG 和工具调用`
- `会处理上下文和记忆`
- `会做工程化部署与评测`
- `能把自己的项目讲得很实`
