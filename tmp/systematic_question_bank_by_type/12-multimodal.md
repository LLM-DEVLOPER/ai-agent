# 多模态与截图理解

重要性：

- 高频程度：中低，但二面深挖常见
- 对应原始题库：[12-multimodal.md](/Users/nako/WebstormProjects/github/thefoxfairy/interview/interview/tmp/raw_question_source_by_type/12-multimodal.md)

## 这一类在考什么

主要在考你是否理解多模态输入会改变索引、检索、评测和解释链路。

## 必会主问题

1. 多模态 RAG 和纯文本 RAG 有什么不同？
2. 图片、截图、图表如何建索引？
3. CLIP 适合解决什么问题？
4. 给定软件截图，如何通过 RAG 帮用户理解界面组件？
5. 多模态用户信息如何存储和使用？

## 高频追问

1. 伪多模态 RAG 和原生多模态 RAG 的区别是什么？
2. 图表类内容怎么评测？
3. 图片理解类任务如何和工具调用结合？

## 标准回答抓手

- 讲清模态表示、索引表示、召回策略
- 截图理解可以拆成：`结构识别 -> 检索 -> 解释`

## 常见失分点

- 把多模态讲成“把图丢给模型”
- 不知道多模态检索和评测怎么做
