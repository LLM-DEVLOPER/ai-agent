重要性：

- 高频程度：最高
- 对应原始题库：[04-memory-context.md](/Users/nako/WebstormProjects/github/thefoxfairy/interview/interview/tmp/raw_question_source_by_type/04-memory-context.md)

## 这一类在考什么

这类问题本质上在考你是否理解：上下文窗口不是记忆系统，长上下文也不是万金油。

## 这一类的核心主线

1. `记忆分层`
2. `上下文裁剪`
3. `写入和召回策略`
4. `多轮状态管理`
5. `成本与效果权衡`

## 必会主问题

1. 短期记忆、长期记忆、工作记忆怎么设计？
2. 长上下文为什么不是越长越好？
3. 多轮对话上下文裁剪怎么做？
4. 上下文压缩、摘要记忆、原始轨迹保留分别适合什么场景？
5. 记忆和 RAG 的边界是什么？
6. 记忆污染、记忆覆盖、旧信息干扰怎么处理？
7. 多轮状态如何在并发下隔离？
8. 长期记忆如何写入、召回和淘汰？

## 高频追问

1. 如何防止无关历史污染当前任务？
2. 用户画像和任务状态是否要分开存？
3. 如何做记忆衰退？
4. 上下文预算有限时优先裁什么？

## 标准回答抓手

- 分清 `会话态 / 用户态 / 任务态`
- 不要把全部历史直接拼进 Prompt
- 讲清 `写入策略、召回策略、淘汰策略、一致性`

## 常见失分点

- 把 memory 等同于聊天记录
- 不知道长期记忆到底存什么
- 不考虑 token 成本和并发隔离
