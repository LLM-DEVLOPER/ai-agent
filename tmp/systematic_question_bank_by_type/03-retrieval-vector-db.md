# 检索、召回与向量库

重要性：

- 高频程度：高
- 对应原始题库：[03-retrieval-vector-db.md](/Users/nako/WebstormProjects/github/thefoxfairy/interview/interview/tmp/raw_question_source_by_type/03-retrieval-vector-db.md)

## 这一类在考什么

面试官想知道你是否理解检索层的工程取舍，而不是只知道“存到向量库里然后搜”。

## 这一类的核心主线

1. `数据规模`
2. `查询模式`
3. `过滤需求`
4. `索引结构`
5. `效果与延迟权衡`

## 必会主问题

1. 向量数据库如何选型？
2. `pgvector`、`FAISS`、`Milvus` 这类方案分别适合什么场景？
3. `IVF` 和 `HNSW` 的区别是什么？
4. BM25 和向量检索各有什么优缺点？
5. 为什么很多业务要做混合检索？
6. embedding 模型应该怎么选？
7. 召回和重排的边界是什么？
8. 检索层怎么做评测？

## 高频追问

1. 检索 latency 高时优先优化哪里？
2. 权限过滤和时间过滤应该落在哪一层？
3. 索引更新频繁时怎么兼顾效果和成本？
4. 知识图谱、图数据库和向量检索的边界是什么？

## 标准回答抓手

- 先讲业务约束，再讲技术选型
- 先说“为什么要混合检索”，再说“怎么实现”
- 评测不要只讲准确率，还要讲延迟和过滤正确性

## 常见失分点

- 只会背库名，不会讲适用场景
- 不知道过滤需求会改变选型
- 不会区分召回和重排职责
