# AI Agent / 大模型应用开发高质量面试题库

整理时间：2026-03-24

题库目标：

- 面向国内公司 `AI Agent 应用开发`、`大模型应用开发`、`LLM/Agent 平台`、`RAG 工程`、`AI 产品/算法` 相关岗位。
- 题目按面试官常见追问路径重新组织，不再按网站堆链接。
- 重点覆盖 2024-2026 中文社区与面经高频方向。

---

## 一、Agent 基础与系统架构

1. 什么是 AI Agent？它和普通 LLM ChatBot 的根本区别是什么？
2. 一个生产级 Agent 的核心组件有哪些？为什么至少需要 Planning、Memory、Tool、Observation、Policy 几层？
3. Agent 和 Workflow 的区别是什么？什么场景该上 Workflow，什么场景必须上 Agent？
4. 单 Agent 和多 Agent 的适用边界是什么？
5. 如果让你手搓 Agent，而不是直接用现成框架，你会怎么回答原因？
6. 一个通用 Agent 框架里，Router、Planner、Executor、Memory、Evaluator 应该怎么拆？
7. DeepResearch、Plan-and-Execute、ReAct、Reflexion 这几类模式分别适合什么任务？
8. 如果业务要求高并发、低延迟、低成本，你会保留哪些 Agent 能力，砍掉哪些能力？
9. Agent 为什么容易“看起来很聪明，但线上很脆”？你会如何解释？
10. 如何设计一个“可观测、可回放、可评估、可上线”的 Agent 系统？

面试官想听什么：

- 你能区分“能力演示”与“生产系统”。
- 你知道 Agent 不是越复杂越好，而是要围绕任务类型设计。
- 你能讲清模块边界、失败恢复、观测与成本。

## 二、Prompt / Tool Calling / MCP

1. Prompt Engineering 的目标是什么？它和“多写几句提示词”有什么区别？
2. 如何让模型稳定输出 JSON、函数参数或 schema？
3. Function Calling 在模型层面通常是怎么学到的？
4. Tool Calling 和 Function Calling 的差别是什么？
5. MCP 是什么？它和 Function Calling 的边界是什么？
6. 如果你自己写过 MCP 工具，工具描述应该怎么写才更利于模型正确调用？
7. Prompt Injection 是什么？最危险的注入路径有哪些？
8. 工具返回脏数据、结构变更、超时、权限错误时怎么兜底？
9. 如何让 Agent 减少无效工具调用和重复推理？
10. Prompt、schema、工具描述、后处理、守卫规则，这五层分别适合解决什么问题？

面试官想听什么：

- 你不仅知道概念，还理解协议边界和失败模式。
- 你能从提示词控制延展到工具治理、风控和降级。
- 你知道模型能调工具，不代表系统就可靠。

## 三、RAG 全链路

1. RAG 想解决的核心问题是什么？
2. RAG 和微调各自适合什么问题，如何做选型？
3. 一个标准 RAG 链路包含哪些模块？
4. 文档切分策略有哪些？什么时候适合定长切分，什么时候需要语义切分？
5. 如何确定 chunk size、chunk overlap、metadata 字段？
6. query rewrite、query expansion、router、rerank 分别解决什么问题？
7. 召回率低时，你会优先检查哪些环节？
8. 检索内容相关但回答仍然不准，原因可能在哪？
9. 如何把业务过滤条件、权限、时效性、用户画像放进检索链路？
10. RAG 怎么处理知识时效性和版本漂移？
11. 什么时候应该从“单路检索”升级到“混合检索”？
12. RAG 线上 bad case 怎么分类？常见类型有哪些？
13. 如果用户问题非常开放，检索目标并不唯一，怎么设计回答链路？
14. 社区内容、企业文档、结构化数据库三种知识源，接入方式有何不同？
15. 如何区分“该改召回”“该改重排”“该改生成”“该改答案约束”？

面试官想听什么：

- 你能讲完整链路，而不是只会背“检索增强生成”四个字。
- 你有排障思维，能从症状拆回模块。
- 你理解业务检索与通用问答检索并不一样。

## 四、记忆、规划与多轮对话

1. Agent 记忆一般如何分层？短期记忆、长期记忆、工作记忆分别怎么用？
2. 多轮对话里为什么历史信息会逐渐衰减？
3. 什么时候应该用总结记忆，什么时候应该保留原始轨迹？
4. 长短期记忆如何配合 RAG 使用？
5. Agent 如何进行任务规划？Planner 失败后怎么恢复？
6. ReAct 的核心流程是什么？优缺点分别是什么？
7. 如果模型规划正确但执行错误，如何定位问题？
8. 如果执行正确但规划路线明显绕远，应该改哪里？
9. 如何减少多轮对话中的上下文膨胀和 token 浪费？
10. 超长上下文和记忆系统到底是什么关系？为什么不是“上下文越长越好”？

面试官想听什么：

- 你理解 memory 不是简单“把聊天记录塞回去”。
- 你知道规划、记忆、上下文窗口、RAG 是互相补位关系。
- 你能讲出 token 成本与效果之间的权衡。

## 五、多 Agent

1. 什么情况下值得上多 Agent？
2. 多 Agent 系统最常见的收益是什么，最常见的问题又是什么？
3. 多 Agent 间状态共享一般怎么做？
4. 多 Agent 通信协议应该如何设计？
5. 如果多个 Agent 给出冲突结果，谁来裁决？
6. 如何避免多 Agent 死循环、相互推诿或重复执行？
7. 多 Agent 如何做成本控制和并发控制？
8. 如果一个 Agent 专门做检索，一个 Agent 专门做生成，一个 Agent 专门做评审，这种拆法什么时候值？
9. 多 Agent 系统如何做日志、回放和观测？
10. 多 Agent 比单 Agent 好在哪里，不好又在哪里？

面试官想听什么：

- 你不会为了“听起来高级”滥用多 Agent。
- 你知道多 Agent 带来的是系统复杂度，不只是能力增强。
- 你能给出清晰的角色边界和裁决机制。

## 六、评测、幻觉与安全

1. 大模型应用评测通常分哪几层？
2. 如何分别评估检索效果、回答效果、工具调用效果、业务效果？
3. 一个 Agent / RAG 项目应该怎么定义 bad case？
4. 幻觉问题有哪些典型来源？
5. 如何治理模型幻觉？
6. 模型复读、循环输出的根源通常是什么？
7. 指令跟随能力一般如何评估？
8. 开放问答、结构化提取、任务执行三类任务的评测方法有什么不同？
9. 如何做离线评测和在线评测联动？
10. 如何保证生成内容合规？内容安全应该卡在哪几层？
11. 如果业务方说“这个 Agent 不靠谱”，你如何把“不靠谱”拆成可验证指标？
12. 如果模型表现退化了，你怎么判断是数据漂移、模型版本变动还是检索库问题？

面试官想听什么：

- 你理解“评测”不是只会说 Rouge、BLEU。
- 你有体系化的 bad case 管理思路。
- 你知道线上稳定性和内容安全同样重要。

## 七、部署、性能与工程化

1. 大模型 API 服务的主要性能瓶颈有哪些？
2. 端到端延迟一般从哪些环节优化？
3. 如何处理高并发下的限流、排队、超时、重试和熔断？
4. Agent 服务高可用一般如何设计？
5. 如果显存不够，常见优化手段有哪些？
6. vLLM、KV Cache、Batching、Streaming 分别解决什么问题？
7. 如何做模型路由，平衡效果、时延和成本？
8. 如何设计一套可观测性指标，覆盖吞吐、时延、成本、错误率、工具成功率？
9. 线上 API 成本不断上涨时，你优先从哪些方向优化？
10. 为什么说“Prompt 写得好”并不能替代工程治理？
11. 生产环境下如何做版本发布、灰度和回滚？
12. 如果某个工具接口质量差，但业务又必须依赖，Agent 侧应该怎么办？

面试官想听什么：

- 你有工程化意识，知道上线系统的现实约束。
- 你知道延迟、成本、稳定性是硬指标。
- 你理解“观测-告警-回滚-复盘”闭环。

## 八、模型基础、训练与微调

1. Transformer 的核心结构是什么？
2. Attention 和 Self-Attention 的区别是什么？
3. 为什么需要多头注意力？
4. 位置编码有什么作用？
5. BERT 和 GPT 的训练目标差异是什么？
6. Prefix LM 和 Causal LM 的区别是什么？
7. 当前主流开源模型体系有哪些？各自生态优势如何？
8. SFT、RLHF、DPO、PPO 分别是什么？
9. LoRA、QLoRA、AdaLoRA、LongLoRA 的适用边界是什么？
10. MoE 相对 Dense 的主要收益和主要难点是什么？
11. 长上下文的一般做法有哪些？
12. embedding 微调一般怎么构造正负样本？
13. 指令微调和标准微调最大的差别是什么？
14. 微调数据集该如何构建与筛选？

面试官想听什么：

- 你至少能把基础原理讲得自洽。

## 九、业务落地与系统设计题

1. 设计一个面向企业知识库问答的 Agent 系统，你的方案是什么？
2. 设计一个内容社区的“种草问答 / 攻略生成 / 审核辅助”Agent，你如何设计？
3. 设计一个游戏客服 Agent，如何接入黑话、FAQ、工单系统与人工兜底？
4. 设计一个 AI 产品，从需求、数据、模型、评测到上线，你怎么推进？
5. 如果老板只给你很少预算，你会优先做 RAG、微调还是 Prompt 工程？
6. 如果业务问“这个 AI 能不能赚钱”，你怎么回答？
7. 用户觉得答案不稳定，你会如何解释并给出改进计划？
8. 一个场景到底适合做问答、推荐、生成、执行 Agent，还是工作流自动化？
9. 如果你的项目用了 Qwen、DeepSeek、GLM 等模型，为什么这么选？
10. 项目里最难的 bad case 是什么？你是怎么定位和修复的？

面试官想听什么：

- 你能把技术方案映射到业务结果。
- 你能在资源有限时做取舍。
- 你说得出效果、成本、时延和风险。

## 十、项目追问与反问模板

高频项目追问：

1. 项目的核心目标是什么？
2. 你在项目里的真实职责是什么？
3. 数据从哪里来，如何清洗和标注？
4. 为什么选这个模型和这个框架？
5. 效果怎么衡量？指标如何定义？
6. 最难的 bad case 是什么？
7. 上线后遇到过什么事故或失败？
8. 如果重做一遍，你会改什么？

高质量反问：

1. 这个岗位当前更偏 Agent 应用、RAG 平台、模型后训练，还是推理工程？
2. 团队现在最关注的是效果、成本、时延，还是业务闭环？
3. 线上系统主要的 bad case 和瓶颈是什么？
4. 团队评估体系是怎么搭的？
5. 这个岗位前三个月希望我解决什么问题？

## 准备建议

### 1. 先准备“骨架”

- Agent 架构
- Tool / MCP / Prompt
- RAG 链路
- 评测体系
- 部署与成本
- 项目复盘

### 2. 再准备“场景”

- 企业知识库
- 内容社区
- 游戏客服
- 审核辅助
- 个人助理 / DeepResearch

### 3. 最后准备“追问”

- 为什么这样选型
- 为什么不用别的方案
- 怎么评估
- 怎么上线
- 怎么处理 bad case

## 高频结论

- `RAG + Agent + 工具调用 + 评测 + 工程化` 是最稳定的高频组合。
- 国内公司面试越来越偏 `系统设计 + 项目追问 + 业务落地`，纯背八股不够。
- 真正能拉开差距的，是你能否把“模型能力”翻译成“上线系统能力”。
