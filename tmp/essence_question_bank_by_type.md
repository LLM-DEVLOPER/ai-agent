# AI Agent / LLM 开发面试精华题库（按类型）

整理时间：2026-03-24

适用岗位：

- `AI Agent 开发`
- `大模型应用开发`
- `RAG 工程`
- `LLM 平台 / Agent 平台`
- `AI Infra / 推理部署`

整理原则：

- 只保留 `100` 篇开发向面经里的高频精华
- 以 `类型` 为主，不按公司堆题
- 每类只保留最值得准备的核心题

---

## 一、Agent 基础类

### 必背题

1. 什么是 `Agent`？它和普通 ChatBot、Workflow 的区别是什么？
2. 一个生产级 Agent 的核心模块有哪些？
3. 单 Agent 和多 Agent 分别适合什么场景？
4. Agent 为什么在线上容易“不稳定”？
5. Agent 和 Workflow 怎么选型？

### 答题抓手

- 先讲定义：`感知 + 规划 + 记忆 + 工具 + 执行`
- 再讲区别：ChatBot 偏问答，Workflow 偏固定流程，Agent 偏动态决策
- 再讲落地：线上最难的是稳定性、观测、容错、成本

### 面试官真正想听

- 你不是只会喊概念
- 你知道 Agent 是系统，不是一个 Prompt
- 你能把能力演示和生产系统分开讲

---

## 二、RAG 类

### 必背题

1. RAG 想解决什么问题？
2. 一个完整的 RAG 链路有哪些模块？
3. chunk size、overlap、metadata 怎么设计？
4. 检索效果不好时怎么排障？
5. 什么时候该改召回，什么时候该改重排，什么时候该改生成？
6. RAG 和微调怎么选？
7. 混合检索为什么比单路检索更稳？

### 答题抓手

- 标准链路：`query 处理 -> 检索 -> 重排 -> 上下文构造 -> 生成 -> 引用/约束`
- 排障顺序：`分块 -> embedding -> 召回 -> rerank -> prompt -> answer constraint`
- 选型核心：`知识更新频繁优先 RAG，行为模式稳定再考虑微调`

### 面试官真正想听

- 你能讲全链路
- 你有模块化排障思维
- 你知道业务知识库和通用问答的差异

---

## 三、记忆 / 上下文工程类

### 必背题

1. Agent 的短期记忆、长期记忆、工作记忆分别怎么设计？
2. 长上下文为什么不是越长越好？
3. 多轮对话的上下文裁剪怎么做？
4. 记忆和 RAG 的边界是什么？
5. 记忆污染、记忆覆盖、旧信息干扰怎么解决？

### 答题抓手

- 短期记忆管当前会话，长期记忆管历史用户画像和事实，工作记忆管任务态
- 不要把全部历史直接塞回上下文，要做 `裁剪 + 摘要 + 检索回写`
- 记忆系统重点讲：`写入策略、召回策略、淘汰策略、一致性`

### 面试官真正想听

- 你知道 memory 不是简单聊天记录拼接
- 你理解 token 成本和效果权衡
- 你能讲清记忆失效的工程问题

---

## 四、工具调用 / Function Calling / MCP 类

### 必背题

1. Function Calling 是什么？它和 Tool Calling 有什么关系？
2. MCP 是什么？和普通函数调用边界在哪里？
3. 工具描述怎么写，模型才更容易正确调用？
4. 工具超时、报错、脏数据、权限错误怎么兜底？
5. 怎么减少无效工具调用和重复调用？

### 答题抓手

- 先讲协议：模型负责选工具，系统负责执行、校验、回传
- 再讲治理：`schema 约束 + 参数校验 + 超时重试 + fallback + 日志`
- MCP 重点讲：`标准化工具接入、可扩展性、跨工具互操作`

### 面试官真正想听

- 你不只会调接口
- 你知道调用成功率是工程问题，不只是模型问题
- 你考虑过异常分支和失败恢复

---

## 五、项目深挖类

### 必背题

1. 你做过的 Agent / RAG 项目核心目标是什么？
2. 你在项目里的真实职责是什么？
3. 为什么选这个模型、框架、向量库？
4. 效果怎么评估？线上指标是什么？
5. 最难的 bad case 是什么？怎么定位和修复？
6. 如果重做一遍，你会改什么？

### 答题抓手

- 用固定结构讲：`业务目标 -> 方案设计 -> 你负责什么 -> 指标 -> bad case -> 优化`
- 不要空讲“做了 RAG”，要说明：`数据、检索、评测、上线`
- 一定准备 `1-2 个失败案例`

### 面试官真正想听

- 你是不是项目真正做事的人
- 你是否具备复盘和优化能力
- 你能不能把“做过”讲成“讲得清”

---

## 六、评测 / 幻觉治理类

### 必背题

1. RAG / Agent 项目怎么做评测？
2. 检索效果、回答效果、工具调用效果怎么分别评估？
3. 幻觉有哪些来源？
4. bad case 怎么分类和闭环？
5. 如果业务说“这个 Agent 不靠谱”，你怎么拆成指标？

### 答题抓手

- 评测拆层：`召回层 -> 生成层 -> 工具层 -> 业务层`
- 幻觉来源通常是：`检索错、上下文不全、Prompt 约束弱、模型乱补`
- 闭环流程：`采样 -> 标注 -> 分类 -> 修复 -> 回归验证`

### 面试官真正想听

- 你有体系化评测思维
- 你不是只会说 Rouge/BLEU
- 你知道线上问题必须落到可验证指标

---

## 七、推理部署 / 性能优化类

### 必背题

1. 大模型应用的端到端延迟主要来自哪些环节？
2. `vLLM`、`KV Cache`、`Batching`、`Streaming` 分别解决什么问题？
3. 显存不够时有哪些常见优化手段？
4. 如何在效果、成本、时延之间做 tradeoff？
5. 为什么线上系统必须做限流、超时、重试、熔断？

### 答题抓手

- 延迟拆解：`检索 -> 工具 -> 模型推理 -> 后处理`
- 优化方向：`模型路由、缓存、批处理、量化、Prompt 裁剪`
- 工程要点：`观测、告警、灰度、回滚`

### 面试官真正想听

- 你理解线上真实约束
- 你知道系统瓶颈不只在模型
- 你能把性能问题拆回具体环节

---

## 八、工程化 / 后端能力类

### 必背题

1. 多用户并发下，Agent 状态怎么隔离？
2. 长任务、异步任务、工具调用链条怎么编排？
3. 如何做日志、观测、回放和故障定位？
4. 如果某个外部工具接口经常不稳定怎么办？
5. 如何保证服务高可用和版本回滚能力？

### 答题抓手

- 重点讲：`状态管理、幂等、超时、任务队列、重试、降级`
- 观测重点：`trace、工具成功率、模型时延、token 成本、bad case`
- 服务化思路：`接口层、编排层、模型层、工具层、日志层`

### 面试官真正想听

- 你有后端工程习惯
- 你知道线上要先活着，再聪明
- 你会排障，不只是会调模型

---

## 九、模型基础 / 微调 / RLHF 类

### 必背题

1. Transformer 的核心结构是什么？
2. 为什么多头注意力有效？
3. LoRA、QLoRA、SFT、DPO、PPO 分别解决什么问题？
4. Qwen、MoE、长上下文这些模型特性会怎么影响应用设计？
5. 什么时候要微调，什么时候只做 Prompt 和 RAG？

### 答题抓手

- 不用讲得太学术，但一定要讲自洽
- 重点是把训练侧能力和应用侧价值连起来
- 最好能说出：`为什么不是所有问题都靠换模型解决`

### 面试官真正想听

- 你有基础，不会只会调 API
- 你知道模型能力和应用工程怎么衔接
- 你不会乱用微调

## 十、手撕 / 编码题类

### 高频方向

1. `LRU`
2. 字符串处理
3. 树的序列化 / 反序列化
4. 并发任务处理
5. 基础检索或数据结构题

### 准备建议

- 不用把 LeetCode 全刷完
- 重点准备：`哈希 + 链表 + 树 + 堆 + 并发基础`
- 开发岗里，代码题通常不是最核心，但会卡基本功

## 十一、只看精华的复习顺序

如果时间不多，直接按这个顺序准备：

1. `Agent 基础`
2. `RAG`
3. `项目深挖`
4. `记忆 / 上下文工程`
5. `工具调用 / MCP / Function Calling`
6. `评测 / 幻觉治理`
7. `推理部署 / 性能优化`
8. `工程化 / 后端能力`
9. `模型基础 / LoRA / DPO`
10. `手撕题`

## 十二、最后结论

这 `100` 篇开发向面经压缩下来，真正的精华主线只有一句话：

- `会讲 Agent`
- `会做 RAG`
- `会处理上下文和记忆`
- `会做工具调用和工程治理`
- `能把项目讲得像真的上过线`

如果这五条能讲稳，开发向 `AI Agent / LLM 应用岗` 的大多数面试都不会偏得太远。
