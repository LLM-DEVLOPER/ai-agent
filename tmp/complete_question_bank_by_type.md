## 一、Agent 基础与系统架构

频次参考：

- `Agent 基础 / 架构 / Workflow`：`92` 次
- 覆盖面经数：`56`

### 核心主问题

1. 什么是 `AI Agent`？和普通聊天机器人有什么根本区别？
2. Agent 和 `Workflow` 的边界是什么？
3. 一个生产级 Agent 至少应该有哪些模块？
4. 单 Agent 和多 Agent 怎么选？
5. Agent 为什么 demo 好看，但线上脆弱？
6. Router、Planner、Executor、Memory、Evaluator 应该怎么拆？
7. 什么时候该做通用 Agent，什么时候该做垂直任务型 Agent？
8. Agent 平台和普通 AI 应用后端有什么差异？

### 高频追问

1. ReAct、Plan-and-Execute、Reflexion、DeepResearch 的差异是什么？
2. Agent 的“自主性”该放到什么程度？
3. Agent 为什么会出现任务漂移和重复思考？
4. Agent 的动作空间为什么必须受约束？
5. 为什么很多业务最后退化成“半 Agent、半 Workflow”？

### 回答抓手

- 定义先讲系统，不要先讲 Prompt
- 强调五件事：`感知、规划、记忆、工具、执行`
- 再讲生产要求：`可观测、可回放、可评估、可降级`
- 最后讲取舍：越通用越复杂，越垂直越稳

### 常见坑

- 把 Agent 讲成“会自动调用工具的大模型”
- 只讲概念，不讲线上失败模式
- 一上来就鼓吹多 Agent

频次参考：

- `RAG / 检索 / 召回 / 重排`：`99` 次
- 覆盖面经数：`55`

### 核心主问题

1. RAG 想解决的核心问题是什么？
2. 一个完整 RAG 链路包含哪些模块？
3. 文档切分怎么做？`chunk size` 和 `overlap` 怎么定？
4. metadata 应该存什么？
5. query rewrite、query expansion、router、rerank 各解决什么问题？
6. RAG 和微调怎么选？
7. 单路检索和混合检索怎么选？
8. 企业文档、社区内容、结构化表数据接入方式有什么不同？
9. RAG 在知识时效性和版本漂移上怎么处理？
10. RAG 线上 bad case 怎么分类？

### 高频追问

1. 为什么检索出来相关内容，回答还是错？
2. 召回率低时优先查什么？
3. 什么时候该换 embedding，什么时候该换 rerank？
4. 检索结果很多但噪声大，怎么收敛？
5. 业务权限、时效性、用户画像怎么接进检索链路？

### 回答抓手

- 固定链路：`query 处理 -> 召回 -> 重排 -> 上下文构造 -> 生成 -> 答案约束`
- 排障路径：`分块 -> 向量化 -> 召回 -> 重排 -> Prompt -> 生成约束`
- 强调 RAG 不是只有“检索增强生成”，而是完整工程链路

### 常见坑

- 只会背 chunk、embedding，不会讲 rerank 和 answer constraint
- 说不清“相关但回答仍错误”的原因
- 不会把 RAG 问题拆回具体模块

3. `IVF` 和 `HNSW` 的区别是什么？

### 核心主问题

1. Prompt Engineering 的核心目标是什么？
2. 用户意图不完整时如何补全？
3. Prompt 如何约束结构化输出？
4. 什么情况下要用 ReAct？
5. Planner 失败时如何恢复？
6. Prompt Injection 是什么？怎么防？
7. Prompt 优化和后处理规则分别该解决什么问题？

### 高频追问

1. 什么时候问题应该交给 Workflow，而不是继续让模型规划？
2. 为什么同一份 Prompt 在线上不稳定？

### 回答抓手

### 常见坑

- 把 Prompt Engineering 讲成经验玄学
- 所有问题都想靠 Prompt 解决
- 不知道 Prompt Injection 的实际风险面

## 十七、总判断

从这 `100` 篇开发向面经里提炼，最重要的不是“你知不知道很多模型名词”，而是：

1. 你能不能把 `Agent` 讲成一个完整系统
2. 你能不能把 `RAG` 讲成一条可排障的工程链路
3. 你能不能讲清 `记忆、上下文、工具调用`
4. 你能不能处理 `评测、幻觉、性能、并发、容错`
5. 你能不能把自己的项目讲得像真的上线过

只要这五条扎实，开发向 `AI Agent / LLM 应用岗` 的主流面试基本都能对上。
